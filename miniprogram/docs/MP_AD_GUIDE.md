# 遗产通 · 微信小程序流量主与广告变现指引

**文档版本**：v1.0  
**适用产品**：遗产通（法律/继承知识类微信小程序）  
**维护说明**：微信广告政策与分成规则以 [微信公众平台](https://mp.weixin.qq.com) 及流量主后台最新公示为准；本文档中的收益区间为行业经验值，实际以结算数据为准。

---

## 1. 微信小程序流量主概述

### 1.1 什么是流量主

**流量主**是微信小程序为开发者提供的官方商业化能力：开发者在小程序内接入微信广告组件，由微信统一撮合广告主投放；用户产生曝光或点击等行为后，平台按规则与开发者结算收益。

### 1.2 基本运作流程

1. 小程序满足开通条件后，在公众平台开通「流量主」。
2. 在「流量主 → 广告管理」中创建广告位，获取各形态的 **广告位 ID**（`unit-id`）。
3. 在页面 WXML 中放置 `<ad>`、`<ad-custom>` 或激励视频等组件，绑定对应 `unit-id`。
4. 用户浏览触发展示；微信侧完成计费、反作弊与结算。
5. 开发者按月查看收入并提现（需完成账户与税务相关配置）。

### 1.3 与遗产通的契合点

遗产通以法条、案例、工具与 AI 辅助为核心，用户停留时间与详情页深度较好，适合在 **非打断主阅读路径** 的位置布置 Banner / 原生模板；在 **明确价值交换** 场景（如导出前观看激励视频）使用激励广告，兼顾体验与 eCPM。

## 2. 开通条件（2025–2026 政策要点）

> 以下归纳自微信对小程序流量主的常见公开要求；细则以提审时平台校验为准。

| 条件项 | 说明 |
|--------|------|
| **累计独立访客（UV）** | 近 30 日累计独立访客 **≥ 500**（历史上曾要求更高门槛，近年已下调；请以开通页提示为准）。 |
| **小程序状态** | 已正式发布上线，非仅体验版。 |
| **合规记录** | 无严重违规、未处于限制期；类目与内容符合平台规范。 |
| **主体类型** | **个人主体与企业主体均可申请**（部分广告形态或行业可能有额外限制，以后台为准）。 |

**运营建议**：在 UV 未达标前，可先完成埋点（如 `wx.reportEvent`）、留存与内容优化；达标后一次性开通并小流量验证再全量。

## 3. 广告形式对比

| 广告类型 | 计费方式 | 单次收益（参考区间） | 用户体验影响 | 推荐位置 |
|----------|----------|----------------------|--------------|----------|
| **Banner 广告** | 多按点击（CPC）为主，兼有关键词/展示维度 | 约 **¥0.2–0.5/次点击**（随行业与时段波动） | 低–中：固定占位，需避免遮挡正文 | 页面底部、列表页脚 |
| **插屏广告（Interstitial）** | 多按曝光计费 | 约 **¥0.3–1.0/次有效曝光** | 中高：全屏打断，需严格频控 | 自然段落结束、会话结束节点 |
| **激励视频** | 按有效观看曝光 | 约 **¥0.5–1.0/次**（通常高于 Banner，**eCPM 潜力最高**） | 中：用户主动选择观看以换取权益 | 导出、高级功能、额外次数前 |
| **原生模板广告** | 多按点击，部分与展示相关 | 与 CPC 及填充相关，需 A/B | 低–中：与信息流融合度好 | 列表中部、卡片之间 |

**说明**：「单次收益」为便于排期的粗算口径，真实收入受 **填充率、用户画像、广告主竞价、季节** 等影响极大，应以流量主后台为准。

## 4. 遗产通推荐广告位规划

结合当前产品结构（首页、法律库、案例库、工具箱、搜索、法条/案例详情、AI 助手、我的），建议如下：

| 优先级 | 页面/场景 | 形态 | 说明 |
|--------|-----------|------|------|
| P0 | **法条详情页底部** | Banner | 阅读完法条后自然视线落点，对正文干扰小。 |
| P0 | **案例详情页中部** | 原生模板（Native） | 嵌入「案情/裁判要旨」段落之间，样式与卡片统一。 |
| P1 | **AI 对话结束后** | 插屏 | 一轮问答完成后的停顿点；建议 **同一用户 30 分钟内最多 1 次**（频控）。 |
| P1 | **工具导出前** | 激励视频 | 用户明确需要「导出/复制长文」等权益时再触发，接受度高。 |
| P2 | **搜索结果列表** | 原生模板 | 穿插在法条/案例/FAQ 等结果之间，注意与结果类型标签区分。 |

**体验原则**：法律类内容以 **可信、可读** 为先；广告密度过高易引发差评与审核风险，建议 RemoteConfig 可一键降级。

## 5. 接入代码示例

### 5.1 基础 Banner（`ad` 组件）

在对应页面 `json` 中声明（若使用插件或特定组件按官方文档）：

```json
{
  "usingComponents": {}
}
```

WXML 示例（`unit-id` 需替换为流量主后台真实 ID）：

```xml
<view class="ad-wrap" wx:if="{{adEnabled}}">
  <ad
    unit-id="adunit-xxxxxxxxxxxxxxxx"
    ad-type="banner"
    ad-theme="white"
    bindload="onAdLoad"
    binderror="onAdError"
  />
</view>
```

### 5.2 原生模板广告

使用 `<ad-custom>` 或平台当前推荐的原生组件形态，绑定 **原生广告位 ID**，并在 `bindload` 中按模板 ID 渲染布局（以微信最新文档为准）。

### 5.3 激励视频（逻辑层）

```javascript
let videoAd = null

function showRewardedIfReady() {
  if (!videoAd) {
    videoAd = wx.createRewardedVideoAd({ adUnitId: 'adunit-yyyyyyyyyyyyyyyy' })
    videoAd.onError((err) => console.error('rewarded error', err))
  }
  videoAd.show().catch(() => {
    videoAd.load().then(() => videoAd.show()).catch(() => {})
  })
}

// 用户点击「观看广告后导出」
showRewardedIfReady()
```

**注意**：激励视频需在用户 **主动点击** 后触发；不得虚假承诺或强制循环。

## 6. RemoteConfig 控制方案（`remote_config.json`）

与《技术设计》中 **AdManager + 远程配置** 思路对齐：将广告开关、位点、频控放在可热更新的 JSON（托管于合法 HTTPS 域名、云开发数据库或 CI 发布物），小程序启动或 `onShow` 时拉取并缓存。

### 6.1 示例 `remote_config.json`

```json
{
  "version": 1,
  "ads": {
    "masterSwitch": true,
    "lawDetailBanner": { "enabled": true, "unitId": "adunit-xxx-banner-law" },
    "caseDetailNative": { "enabled": true, "unitId": "adunit-xxx-native-case" },
    "aiInterstitial": {
      "enabled": true,
      "unitId": "adunit-xxx-interstitial-ai",
      "minIntervalSec": 1800
    },
    "toolExportRewarded": { "enabled": true, "unitId": "adunit-xxx-rewarded-tool" },
    "searchNative": { "enabled": false, "unitId": "adunit-xxx-native-search" },
    "hideDuringReview": true
  }
}
```

### 6.2 控制逻辑要点

| 配置项 | 用途 |
|--------|------|
| `masterSwitch` | 总开关：审核期、舆情期可 false。 |
| `hideDuringReview` | 与提审版本号或时间窗联动，自动关闭广告展示。 |
| `minIntervalSec` | 插屏/激励等 **频控**（如 AI 结束插屏 1800s = 30 分钟）。 |
| 分广告位 `enabled` | 按页灰度，便于 A/B 与问题回滚。 |

**安全提示**：`remote_config.json` 所在域名须加入小程序 **downloadFile / request 合法域名**；建议 CDN 短缓存 + 本地 `Storage` 兜底，避免频繁请求失败导致页面异常。

## 7. 收入预估表（示意）

假设：**混合形态**、**每 DAU 日均有效广告收益约 ¥0.03–0.08**（法律类偏低、一线城市略高；含 Banner + 少量原生/插屏/激励的粗算）。以下为 **区间估算**，用于立项与排期，**非承诺**。

| DAU | 预估日收入（元） | 预估月收入（元，按 30 天） |
|-----|------------------|----------------------------|
| 500 | 15 – 40 | 450 – 1,200 |
| 1,000 | 30 – 80 | 900 – 2,400 |
| 5,000 | 150 – 400 | 4,500 – 12,000 |
| 10,000 | 300 – 800 | 9,000 – 24,000 |

优化方向：提升 **激励视频占比**（在合规前提下）、优化 **原生位点击率**、保证 **案例/搜索停留时长**。

## 8. 分成规则（摘要）

微信流量主收入为平台与开发者 **按比例分成**。你方提供的内部口径可归纳为：

- **日流水（或对应结算口径）在 10 万元以内部分**：开发者约 **50%**；
- **超出部分**：开发者约 **30%**。

**重要**：具体阶梯、含税、结算周期与「流水」定义以 **微信流量主协议及后台账单** 为准；上线前请由财务或法务核对最新版协议。

## 9. 合规注意事项（法律类小程序）

| 领域 | 建议 |
|------|------|
| **内容与广告** | 不得将广告伪装成法条、裁判文书或政府公告；广告需有清晰「广告」标识（按微信组件规范）。 |
| **法律服务边界** | 小程序内已有「仅供参考、非法律意见」类声明时，广告不得暗示「律师代理」「包赢」等违规承诺。 |
| **个人信息** | 广告 SDK 涉及的采集说明需在隐私政策中覆盖（按平台模板更新）。 |
| **未成年人** | 若用户画像含未成年，需遵守未成年人保护及广告适龄要求。 |
| **审核与舆情** | 重大节点（法考、热点案件）可临时通过 RemoteConfig 关闭或降级广告，降低误触风险。 |

## 10. 行动清单（Phase 0–3）

| 阶段 | 时间线（建议） | 关键任务 |
|------|----------------|----------|
| **Phase 0** | 第 1–2 周 | 确认类目与资质；完成隐私政策与免责声明；埋点验证 DAU/UV 路径。 |

| **Phase 3** | 第 9 周起 | 工具导出激励视频；A/B 实验与位置优化；月度复盘与合规巡检。 |
