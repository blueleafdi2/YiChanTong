## 3. 应用宝上架详细步骤

### 3.1 平台概况

| 项目 | 说明 |
|------|------|
| 覆盖率 | 约 25% 安卓用户 |
| 优势 | 个人开发者不强制软著，审核较快（1–3 天） |
| 费用 | 免费 |
| 官网 | https://app.open.qq.com |

### 3.2 注册与实名认证

1. 打开 https://app.open.qq.com
2. 点击「注册」→ 使用 QQ 或微信登录
3. 选择「个人开发者」
4. 实名认证：
   - 姓名：王迪（与身份证一致）
   - 身份证号
   - 手机号：18028748804
   - 完成人脸识别
5. 等待认证通过（通常即时）

### 3.3 创建应用并上传

1. 进入「应用管理」→「发布应用」
2. 上传 APK：`遗产通-v4.5.0-release-signed.apk`
3. 填写应用信息：

| 字段 | 填写内容 |
|------|----------|
| 应用名称 | 遗产通 |
| 一句话简介 | 中国遗产继承法律百科 · 390+真实判例 · AI智能解读 · 25专题125术语 |
| 详细描述 | 使用第 2.3 节完整描述模板 |
| 应用分类 | 工具 → 法律（或 教育 → 工具书） |
| 内容分级 | 所有人 |
| 隐私政策 URL | https://blueleafdi2.github.io/YiChanTong/ |
| 开发者名称 | 王迪 |

4. 上传截图：至少 4 张，建议 6 张（见 1.5 节）
5. 确认信息无误后提交审核

### 3.4 应用宝审核要点

- 应用简介控制在 500 字符内
- 截图需为真实应用界面，不得造假
- 隐私政策 URL 必须可访问
- 描述中避免「第一」「最好」等绝对化用语
- 法律类应用需有免责声明

## 7. 常见拒绝原因与解决方案

### 7.1 隐私政策相关

| 拒绝原因 | 解决方案 |
|----------|----------|
| 未提供隐私政策 | 使用已部署 URL：https://blueleafdi2.github.io/YiChanTong/ |
| 隐私政策无法访问 | 检查 URL 是否可公网访问，避免 404 |
| 隐私政策内容不完整 | 补充信息收集、使用、存储、第三方服务、联系方式等章节 |

### 7.2 资质相关

| 拒绝原因 | 解决方案 |
|----------|----------|
| 缺少软件著作权 | 华为/小米/OPPO/vivo 必需；可先上应用宝、酷安 |
| 软著信息与应用不符 | 确保软著中软件名称、版本号与实际上架应用一致 |
| 实名认证未通过 | 确保姓名、身份证、人脸与本人一致 |

### 7.3 应用内容相关

| 拒绝原因 | 解决方案 |
|----------|----------|
| 截图不清晰或造假 | 使用真实应用截图，分辨率符合要求 |
| 描述含违规用语 | 避免「第一」「最好」「 guaranteed」等绝对化表述 |
| 法律类应用无免责 | 在描述与 App 内明确标注「仅供参考，不构成法律意见」 |
| 应用与描述不符 | 确保功能描述与实际功能一致 |

### 7.4 技术相关

| 拒绝原因 | 解决方案 |
|----------|----------|
| 应用崩溃或无法启动 | 在主流机型上充分测试后再提交 |
| 权限申请不合理 | 仅申请必要权限，并在隐私政策中说明用途 |
| 签名问题 | 使用正式签名 APK，勿用 debug 包上架 |

### 7.5 应用宝专项

| 拒绝原因 | 解决方案 |
|----------|----------|
| 应用简介超 500 字 | 使用第 2.2 节 60–500 字符版本 |
| 分类选择不当 | 选择「工具 → 法律」或「教育 → 工具书」 |
| 重复应用 | 若已有同名应用，需提供软著等证明为原创 |

## 8. 商业化与数据分析

### 8.1 已内置能力（v4.5.0）

| 能力 | 用途 |
|------|------|
| **Analytics 追踪** | 统计页面访问、功能使用、用户路径，支持后续产品与运营决策 |
| **RemoteConfig** | 远程开关功能、调整文案、灰度发布，无需发版即可调整 |
| **Fragment 状态保持** | 提升体验，降低因体验差导致的流失 |

### 8.2 商业化方向建议

1. **免费 + 增值**：基础功能免费，高级功能（如 AI 深度分析、导出报告）付费
2. **广告**：在非核心页面接入信息流或激励视频（需符合各商店广告政策）
3. **内容付费**：专题课程、专家解读等
4. **B2B**：向律所、法务团队提供企业版或 API

### 8.3 数据驱动迭代

- 利用 Analytics 识别高价值功能与流失节点
- 用 RemoteConfig 做 A/B 测试与功能灰度
- 结合应用商店评论与反馈优化版本规划

## 9. 合规与风险规避

### 9.1 风险与现状

| 风险点 | 现状 | 风险等级 | 建议 |
|--------|------|----------|------|
| 法律内容准确性 | 引用《民法典》原文 | 低 | 保持法条编号与出处 |
| 律师执业资质 | 未提供法律服务 | 低 | App 内注明「仅供参考」 |
| 隐私合规 | 本地存储为主 | 低 | 隐私政策已部署 |
| AI 生成内容 | 有免责提示 | 中低 | 在 AI 回答中加提示 |
| 内容许可 | 法律法规无版权 | 低 | 案例来源可标注 |

### 9.2 已采取的合规措施

1. App 内免责声明：「本 App 内容仅供参考学习，不构成专业法律意见」
2. 法条引用公开法律原文
3. 案例来源于公开裁判文书
4. 无强制注册登录，降低隐私风险
5. 数据本地存储，不涉及数据出境
